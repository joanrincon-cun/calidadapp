Propósito del aplicativo

Esta app demuestra ingeniería web con frontend público, backend mínimo y BD NoSQL (LowDB), integrando un modelo 0–5 con pesos y simulación PUB/SUB.

Objetivo general

Implementar un prototipo funcional que evidencie conceptos clave de arquitectura web, persistencia NoSQL, patrón PUB/SUB y despliegue web.

Objetivos específicos

  • Construir frontend público y responsivo.
  • Exponer API REST mínima con almacenamiento NoSQL (JSON).
  • Modelar y calcular un índice 0–5 con pesos.
  • Documentar arquitectura, PUB/SUB y contenedores/nube.
  • Ejecutar 6 casos de prueba con evidencias.

Nombre de la app

Calidad App

Arquitectura (Frontend + Backend + BD NoSQL)

Frontend estático (GitHub Pages) + Backend Express (Replit) + BD NoSQL (LowDB/JSON). Comunicación por REST (fetch).

  • Frontend: HTML/CSS/JS, público.
  • Backend: Node/Express con endpoints /api/items.
  • NoSQL: LowDB sobre archivo db.json.

PUB/SUB (concepto + diagrama + demo)

Usamos un EventBus simple en el frontend: los productores “publican” eventos y los consumidores se “suscriben”.

// En app.js: window.bus.publish('evento', data); window.bus.subscribe('evento', cb);

Contenedores y nube

Se propone Dockerizar el backend (imagen Node) y desplegarlo en un servicio gestionado. Para el prototipo, se muestra el diseño (diagrama) y la receta Docker.

# Dockerfile (backend)
FROM node:20-alpine
WORKDIR /app
COPY package*.json ./
RUN npm ci --omit=dev
COPY . .
EXPOSE 3000
CMD ["node", "index.js"]

Adjuntar diagrama: Dev → Docker → Cloud (puerto 3000) → Frontend llama /api.
